Then ask the user which feature they want to work on (suggest the top item from "Up Next" if they don't specify).

Once confirmed:
1. Ask any clarifying questions needed to scope the work — keep it tight, don't over-plan
2. Outline what you're going to build (a few bullets, no essay)
3. Create a new git branch named `feature/<slugified-feature-name>` and switch to it
4. Start building it

After the feature is working and tested, remind the user to run `/ship-feature`.
