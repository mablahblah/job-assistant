---
name: ux-flow
description: Walk through UX design decisions, user flows, or interface changes collaboratively. Never assume the user knows what a component, pattern, or behaviour does — explain it, then ask.
---

When working on anything UX-related (new screens, interactions, flows, component behaviour, layout decisions):

1. Never assume the user understands a pattern or component — briefly explain what it is/does before proposing it
2. Walk through the experience step by step from the user's perspective, not the implementation perspective
3. At each decision point, present the options with a plain-language explanation of the tradeoff, then ask which direction they want to go
4. Don't proceed to implementation until the flow is agreed on
5. If a decision has downstream consequences (e.g. affects another screen or state), flag it before moving on
